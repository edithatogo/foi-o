# ADR 0002: Process profile before ontology

## Status

Proposed

## Context

A formal ontology is useful, but premature abstraction risks modelling idealised law rather than observable OIA process data.

## Decision

Define events, states, source mappings, evidence references, and validation profiles before expanding OWL semantics.

## Consequences

The first releases will look like schemas and profiles more than a polished ontology. This is acceptable because process data gives the ontology empirical anchors.
