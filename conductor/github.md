## Linkage Rules

- Each Conductor track should have one GitHub issue labelled `conductor` and `track`.
- The GitHub issue body must reference the local `conductor/tracks/<track_id>/` directory.
- Pull requests must reference the Conductor track ID, local track path, GitHub issue, and project.
- GitHub Project status is a coordination layer; local `plan.md` remains the detailed execution source of truth.
- When a track is archived locally, update the linked GitHub issue and project item in the same closeout.
