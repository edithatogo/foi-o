# Suggested issue backlog

## Issue 1 — Create standalone repo skeleton

Create `foi-o-nz` as a standalone repo with docs, schemas, ontology, vocab, SHACL, mappings, examples, CI, and tests.

## Issue 2 — Define stable namespace and versioning policy

Choose namespace, schema `$id` pattern, release tags, and deprecation policy.

## Issue 3 — Map FYI/Alaveteli source states

Create a source-preserving mapping from FYI/Alaveteli state labels to FOI-O NZ normalised states.

## Issue 4 — Build 100-request gold set

Manually annotate source state, normalised state, key events, outcome, confidence, and evidence references.

## Issue 5 — Validate event schema on examples

Write examples for extension, transfer, clarification, decision communicated, release, refusal, complaint, and closure.

## Issue 6 — Add SHACL certification boundary

Ensure decision-like events require human certification and agent-generated events cannot be marked final.

## Issue 7 — Map PSC reporting metrics

Map PSC OIA metrics to event-log fields and flag metrics not derivable from public FYI data.

## Issue 8 — Add legal-source versioning

Integrate NZ Legislation API identifiers into statutory references.

## Issue 9 — Define future MCP server contract

Define read-only resources and non-dispositive tools. Add tests for unsafe tool descriptions.

## Issue 10 — Write short methods paper

Draft a concept/methods paper: “An agent-facing process ontology for New Zealand Official Information Act administration.”
